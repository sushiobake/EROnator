# UI 目標・将来方針

**更新日: 2026-02-15**

PC・スマホ・タブレット問わず快適にプレイできるUIを目指す。幅に依存せず整った見た目を実現する。

---

## 最終的なUI目標（方針）

### レスポンシブの最適解

**ブレークポイントを 1〜2 箇所に絞り、それ以外は Fluid / Intrinsic に寄せる。**

- ブレークポイントを増やすと機種ごとの細かい違いが増え、端数解像度（375px / 414px / 768.5px など）で不揃いになりやすい。
- Fluid（連続的な変化）と Intrinsic（コンテンツに応じた自然なレイアウト）を主体にし、ブレークポイントは「本当に必要な構造の切り替え」だけに使う。
- 優秀なサイト（Stripe, Linear, Vercel など）もこの方針で、機種差を抑えている。

### 具体的な手法

| 手法 | 説明 |
|------|------|
| **Fluid typography** | `clamp()` でフォントサイズを最小〜最大の範囲で連続変化させる |
| **Fluid spacing** | 余白も `clamp()` や相対単位で幅に応じて変化させる |
| **Intrinsic layout** | Flexbox / Grid の `flex-wrap` や `minmax()` で、幅に応じて自動折り返し。ブレークポイント不要 |
| **Container queries** | ビューポート幅ではなく「親コンテナ幅」に反応させ、機種に依存しにくくする |
| **最小限のブレークポイント** | レイアウトの構造が根本的に変わる箇所（例: ナビのハンバーガー化）のみに使用 |
