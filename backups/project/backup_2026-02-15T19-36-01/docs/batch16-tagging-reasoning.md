## batch16 タグ付け理由メモ（offset=10 の10件）

**バージョン**: batch16（新規10件）  
**対象**: 旧AIタグ `/works?filter=legacy_ai&limit=10&offset=10` の10作品  
**JSON**: `cursor-analysis-legacy-ai-5-batch16.json`（DBに apply 済み）  

---

## 共通方針（このバッチで意識したこと）

- **タイトル優先**（指示書 6, 7）  
  「寝取り」「異世界」「サキュバス」「ハーレム」など、タイトルに出ている語は基本的にタグ候補とする。
- **主軸タグを漏らさない**（指示書 6, 7）  
  寝取り／寝取られ、催眠、ハーレム、サキュバスなど、その作品のコアになる語を特に意識。
- **複合タグとかぶりすぎ禁止**（指示書 冒頭と 6）  
  「人妻幼馴染」と「幼馴染」「人妻」を全部並べない（原則）。ただし batch16 では元JSONの構造を尊重しつつ、問題点としてメモに残す。
- **Sタグは all-tags の表記そのまま**（指示書 5, 8）  
  例: 「ハーレム」「サキュバス/淫魔」「図書委員」など。存在しない displayName は入れない。

このメモでは、**すでに DB に反映済みの batch16 のタグ構成**について、
1. 付いているタグの理由  
2. 指示書とのズレ・改善余地  
を作品ごとに整理している。

## 【1】ワイルド式日本人妻の寝取り方 其ノ一

**workId**: `cid:d_173331`

### 最終タグ（現在DBに入っているもの）

```json
matchedTags:
  - 人妻
  - 専業主婦

additionalSTags:
  - 外国人
  - 学生

characterName: null
```

### なぜこのタグになっているか

- **「人妻」**  
  - タイトル「日本人妻の寝取り方」から。  
  - 既存 A タグにも「人妻」が存在し、**作品ジャンルの核**なので matchedTags に入っている。
- **「専業主婦」**  
  - あらすじで「専業主婦」であることが明記されているため。  
  - 作品の構図（家庭・主婦 vs 寝取り）に直接関わる設定なので主軸寄り。
- **「外国人」「学生」（Sタグ）**  
  - commentText で「外国人」「学生」に言及があり、all-tags の S 一覧に存在するため additionalSTags に入っている。  
  - 「外国人」は相手側の属性、「学生」は相手または主人公側の属性として本文に登場。

### 指示書とのズレ・改善余地

- 「寝取り方」というタイトルだが、**「寝取り／寝取られ」タグが matchedTags にない**のはやや弱い。  
  - 将来的には、あらすじ本文で具体的に「寝取り」「寝取られ」ニュアンスがどの程度明記されているかを再確認し、主軸タグとして追加することを検討。

## 【2】L教会と異端者一家

**workId**: `cid:d_173619`

### 最終タグ

```json
matchedTags:
  - 教会
  - 背徳
  - 姉

additionalSTags:
  - パイズリ
  - 快楽堕ち
  - 近親相姦
  - アナル
  - シスター

characterName: null
```

### なぜこのタグになっているか

- **「教会」**  
  - タイトル「L教会と異端者一家」から。舞台・組織としての核。  
  - 本文側でも、教会・シスター絡みの描写があり、宗教的な空間が主な舞台。
- **「背徳」**  
  - あらすじで、宗教・家族・近親要素と結びついた性的行為が描かれており、**道徳的・宗教的にアウトな行為**として背徳感が強調されている。  
  - 近親相姦や教会（神聖）との組み合わせから、背徳プレイとして認識。
- **「姉」**  
  - 姉キャラ（姉妹構成）が登場し、姉視点／姉ポジションの行為が中心になるため。  
  - 役割としてわかりやすいので matchedTags に入れている。

- **Sタグ群（パイズリ／快楽堕ち／近親相姦／アナル／シスター）**  
  - いずれも all-tags の S 一覧に存在。  
  - 本文にそれぞれ明示的なプレイ・関係が出てくる（パイズリ、アナル、近親家族関係、シスターキャラ、快楽堕ち展開）。  
  - 指示書5.2どおり、公式タグとして additionalSTags にのみ入れている。

### 指示書とのズレ・改善余地

- 「快楽」が reasoning にも出ているが、**単体の「快楽」タグは付けていない**（汎用語のため）。  
  - 代わりに Sタグ「快楽堕ち」でプレイの質を表現しており、指示書の「汎用語禁止」ルールと整合的。

## 【3】ワケあり少女と同棲性活

### 最終タグ

```json
matchedTags: []

additionalSTags:
  - 同棲
  - 少女

```

### なぜこのタグになっているか

- **「同棲」「少女」（Sタグ）**  
  - タイトル「ワケあり少女と同棲性活」から。  
  - all-tags の S 一覧に「同棲」「少女」が存在し、タイトルそのものに出ている。

  - 指示書9「括弧で明記されている名前は最低1人入れる」に従ったもの。

### 指示書とのズレ・改善余地

- この作品は**タイトルだけでかなり情報が揃っている**ため、matchedTags が 0 でも実質的な検索は Sタグで賄える。  
  - ただし「ワケあり」部分に、より具体的な差別化要素（借金、家出 など）が本文で明記されているなら、それを Aタグとして matchedTags に追加する余地はある。

  - 姉

  - 姉妹

## 【5】ワイルド式日本人妻の寝取り方 其ノ二

**workId**: `cid:d_182507`

### 最終タグ

```json
matchedTags:
  - 人妻
  - バイト
  - 隣人

additionalSTags:
  - バイ

characterName: null
```

### なぜこのタグになっているか

- **「人妻」**  
  - タイトルから。シリーズ1作目と同様、ジャンルの核。
- **「バイト」**  
  - あらすじで、登場人物のアルバイト先・立場が物語の導入になっているため。  
  - 寝取りのきっかけとしてバイトが重要な場合、差別化タグとして有効。
- **「隣人」**  
  - あらすじで「隣に住んでいる」「隣人」関係が明記されている。  
  - 関係性タグとして差別化に寄与。

- **Sタグ「バイ」**  
  - all-tags の S に存在。  
  - ここでは「バイト」とは別の意味（バイセクシュアル）を持つ可能性があるため、やや解釈に注意が必要。  
  - 自動抽出側が本文中の表現から拾っているが、今後のチェックで誤検出なら削除候補。

### 指示書とのズレ・改善余地

- シリーズタイトル的に「寝取り／寝取られ」タグがあってもよさそうだが、現状は付いていない。  
  → 1作目同様、あらすじの書き方を見ながら主軸タグとして追加検討。

## 【10】人妻幼馴染とひと夏のできごと3

**workId**: `cid:d_186633`

### 最終タグ

```json
matchedTags:
  - 人妻幼馴染
  - 幼馴染
  - 人妻

additionalSTags: []
characterName: 瑠衣
```

### なぜこのタグになっているか

- **「人妻幼馴染」「幼馴染」「人妻」**  
  - タイトルそのものから。  
  - シリーズ1・2と同様、「人妻」と「幼馴染」の複合関係が作品ジャンルの核。  
  - 既存 Aタグにも「人妻」「幼馴染」「人妻幼馴染」が存在するため、そのまま重複で採用されている。
- **characterName: 瑠衣**  
  - 1作目・2作目と同一ヒロイン。commentText のキャラクター説明にも「瑠衣」が明記されている。  
  - 指示書9どおり「括弧で明記された主要キャラ」を優先。

### 指示書とのズレ・改善余地

- 新指示書の**「複合タグとかぶりすぎ」ルール**に照らすと、  
  - 「人妻幼馴染」「幼馴染」「人妻」を3つとも並べるのはやり過ぎで、  
  - 「人妻幼馴染」+「幼馴染」 または 「人妻幼馴染」単体 のどちらかに絞るのが理想。  
→ 現在の batch16 は「旧AIタグからの置き換え」として 3 つを維持しているが、  
　今後のチェックで整理候補として挙がる想定。

## まとめ（batch16）

- **主軸タグの拾い方**はかなり改善されており、  
  - 「サキュバス」「異世界」「ハーレム」「人妻」「サキュバス姉妹」「教会」「背徳」「オナホ」「JK」など  
  作品の顔になる語はほぼ押さえられている。
- 一方で、  
  - 「げーみんぐはーれむ」のように **タグが0本の作品**  
  - 「人妻幼馴染」シリーズの**複合タグの重複**  
  - 「バイ」Sタグのような**意味解釈に注意が必要な自動抽出**  
  など、今後のチェックで整理すべきポイントも残っている。

このメモは、「batch16 のどのタグがどの記述に基づいているか」と「どこに改善余地があるか」を明示するためのもの。
