# 簡単デプロイガイド

審査通過後、サービスを運用する際の簡単なワークフローです。

## 基本の流れ

1. **ローカルで開発・テスト**（いつも通り）
2. **実装ボタンを押す**（`npm run deploy:prod`）
3. **完了！** 本番環境に反映される

## 使い方

### ステップ1: 開発用ブランチに切り替える（初回のみ）

```powershell
# 開発用ブランチを作る（初回のみ）
git checkout -b develop
git push -u origin develop
```

### ステップ2: ローカルで開発・テスト

いつも通り、ローカルで開発・テストします：

```powershell
# ローカルサーバーを起動
npm run dev

# ブラウザで http://localhost:3000 を開く
# 変更を確認・テスト
```

### ステップ3: 変更を保存（developブランチに）

```powershell
# 変更を保存
git add .
git commit -m "変更内容の説明"
git push origin develop
```

### ステップ4: 実装ボタンを押す

```powershell
# これだけ！本番環境に反映されます
npm run deploy:prod
```

スクリプトが自動的に：
1. developブランチの変更を確認
2. mainブランチにマージ
3. 本番環境にデプロイ
4. developブランチに戻る

## よくある質問

### Q: 実装ボタンを押したら、何が起こるの？

A: 
1. developブランチの変更がmainブランチにマージされます
2. Vercelが自動的に本番環境（`https://eronator.vercel.app`）を更新します
3. 数分で本番環境に反映されます

### Q: 間違えて実装ボタンを押してしまった

A: 
- スクリプトは最終確認を求めます（"yes"と入力しないと実行されません）
- もし実行してしまった場合、Vercelダッシュボードで前のデプロイに戻せます

### Q: ローカルでテストする方法は？

A: 
- いつも通り `npm run dev` を実行
- `http://localhost:3000` で確認
- 変更はローカルのみなので、本番環境には影響しません

### Q: 複数の変更を一度に実装したい

A: 
- 複数の変更をdevelopブランチにコミット
- 最後に `npm run deploy:prod` を1回実行するだけ
- すべての変更が一度に本番環境に反映されます

## 安全のための確認事項

実装ボタンを押す前に：

- [ ] ローカルで動作確認した
- [ ] エラーが発生していない
- [ ] 変更内容を確認した

## まとめ

- **ローカル開発**: いつも通り `npm run dev`
- **実装**: `npm run deploy:prod` を実行
- **完了**: 本番環境に自動反映

これで、簡単に本番環境に反映できます！
